🚀 Go-Live Checklist — น้องใจดี

สิ่งที่ต้องทำก่อนเปิดใช้งานจริงทั้ง LINE OA และ Facebook Messenger

⚠️ ขั้นตอนสำคัญ: ต้องทำให้ครบก่อน Go-Live เพราะถ้าข้ามขั้นตอน บอทอาจส่งข้อความผิด หรือลูกค้าเห็นข้อความ "กำลังทดสอบ"

1 ทั้ง 2 Platform — ทำก่อนเลย

🔴 ลบข้อความ "กำลังทดสอบ" ออกจาก Greeting
ตอนนี้ greeting card มีข้อความ ( ขณะนี้อยู่ระหว่างการทดสอบระบบตอบกลับอัตโนมัติค่ะ )
ต้องแก้ใน 2 ที่:
• lib/flex-cards.ts บรรทัด 22 — LINE OA Flex Card
• app/api/fb-webhook/route.ts บรรทัด 275 — Facebook Quick Reply text
เปลี่ยนเป็นข้อความต้อนรับจริง เช่น "ยินดีต้อนรับสู่ Spender Club ค่ะ สอบถามได้เลยนะคะ 😊"
🔴 ตรวจ Google Sheet ครบถ้วน
ตรวจสอบว่า Sheet มีข้อมูลครบก่อน Go-Live:
• สินค้าทุกรุ่นที่ขายอยู่ (ชื่อ, ราคา, สเปค, ลิงก์สินค้า)
• ข้อมูลการสั่งซื้อ (ช่องทาง, การชำระเงิน, การจัดส่ง)
• ข้อมูลการรับประกันและบริการหลังการขาย
• ข้อมูลติดต่อ (เบอร์โทร, LINE, ที่อยู่, เวลาทำการ)
• คำถามที่ลูกค้าถามบ่อย (ดูจาก /admin/unanswered หลัง Go-Live)
🔴 ตรวจ Environment Variables ใน Vercel ครบทุกตัว
ไปที่ Vercel → Settings → Environment Variables ตรวจว่ามีครบ:
Variableได้จากไหนสถานะ
LINE_CHANNEL_ACCESS_TOKENLINE Developers Consoleจำเป็น
LINE_CHANNEL_SECRETLINE Developers Consoleจำเป็น
GEMINI_API_KEYGoogle AI Studioจำเป็น
SHEET_CSV_URLGoogle Sheet → Share → Publish as CSVจำเป็น
UPSTASH_REDIS_REST_URLUpstash Dashboardจำเป็น
UPSTASH_REDIS_REST_TOKENUpstash Dashboardจำเป็น
FACEBOOK_PAGE_ACCESS_TOKENFacebook Developersจำเป็น
FACEBOOK_APP_SECRETFacebook Developersจำเป็น
FACEBOOK_VERIFY_TOKENกำหนดเองตอนตั้ง Webhookจำเป็น
ADMIN_GROUP_IDLINE Group ID ของแอดมินแนะนำ
ADMIN_SCHEDULE_PASSWORDกำหนดเองตั้งแล้ว
SERPER_API_KEYserper.dev (ถ้าใช้ Web Search)ถ้าใช้
ตั้งเวลาเปิด/ปิดบอทจริงใน /admin/schedule
เปิด https://line-bot-ai-theta.vercel.app/admin/schedule
ตั้งกฎให้ตรงกับเวลาทำการจริงของร้าน เช่น ปิดบอทนอกเวลาทำการ 18:00–07:59

2 LINE OA — ตั้งค่าใน LINE Developers Console

🔴 ตรวจ Webhook URL ใน LINE Developers Console
ไปที่ developers.line.biz → เลือก Channel → Messaging API
Webhook URL ต้องเป็น:
https://line-bot-ai-theta.vercel.app/api/line-webhook
• กด Verify → ต้องขึ้น "Success"
• Use webhook = ON
🔴 ปิด Auto-reply และ Greeting Message ของ LINE เอง
ใน LINE Developers Console → Messaging API:
• Auto-reply messages → Disabled
• Greeting messages → Disabled

หรือไปที่ LINE Official Account Manager (OA Manager):
• การตั้งค่า → ตอบกลับข้อความ → ปิด "ตอบกลับอัตโนมัติ" และ "ข้อความทักทาย" ของ LINE เอง
⚠️ ถ้าไม่ปิด ลูกค้าจะได้รับ 2 ข้อความ (ของ LINE + ของบอท)
ตั้งค่า ADMIN_GROUP_ID สำหรับ Smart Handoff
เมื่อบอทส่งต่อแอดมิน ระบบจะแจ้งเตือนไปยัง LINE Group ที่กำหนด
วิธีหา Group ID:
1
เพิ่มบอท (@SpenderClub) เข้า LINE Group แอดมิน
2
ดู Vercel Logs หลังมีข้อความใน Group → จะเห็น groupId ใน log
3
นำ groupId ไปใส่ใน Vercel → ADMIN_GROUP_ID
ทดสอบ LINE OA ครั้งสุดท้ายก่อน Go-Live
• ส่งข้อความครั้งแรก → ต้องได้รับ Greeting Card (ไม่มีข้อความ "ทดสอบ")
• ถามเรื่องสินค้า → บอทตอบถูกต้อง
• ส่งรูปสินค้า → บอทแสดง Intent Card
• พิมพ์ "ติดต่อแอดมิน" → บอทส่งต่อ + แจ้ง Group แอดมิน
• ทดสอบช่วงเวลาปิดบอท → บอทไม่ตอบ

3 Facebook Messenger — ตั้งค่าใน Facebook Developers

🔴 ตรวจ Webhook ใน Facebook Developers Console
ไปที่ developers.facebook.com → App → Messenger → Webhooks
Callback URL:
https://line-bot-ai-theta.vercel.app/api/fb-webhook
Subscribed Fields ต้องติ๊ก:
• messages
• messaging_postbacks
• messaging_optins (ถ้าใช้)
กด Verify and Save → ต้องสำเร็จ
🔴 ตรวจสอบ Facebook App Permissions
Facebook App ต้องมี Permission:
• pages_messaging — ส่งข้อความจาก Page
• pages_read_engagement — รับข้อความ

ถ้า App ยังเป็น Development Mode:
บอทตอบได้เฉพาะคนที่เป็น Admin/Tester ของ App เท่านั้น
→ ต้อง Submit App for Review เพื่อให้ลูกค้าทั่วไปใช้งานได้

วิธี Submit for Review:
App Review → Permissions and Features → ขอ pages_messaging → กรอกข้อมูล + วิดีโอสาธิต
🔴 ปิด Auto-reply ของ Facebook Page เอง
ไปที่ Facebook Page → Inbox → Automated Responses
• ปิด Instant Reply (ถ้าเปิดไว้)
• ปิด Away Message (ถ้าเปิดไว้)
⚠️ ถ้าไม่ปิด ลูกค้าจะได้รับ 2 ข้อความ (ของ Facebook Page + ของบอท)
ตั้ง Get Started Button (ปุ่มเริ่มต้น)
ปุ่ม "Get Started" ของ Facebook Messenger ส่ง Postback เมื่อลูกค้าเปิด chat ครั้งแรก
ถ้ายังไม่ได้ตั้ง สามารถตั้งผ่าน Graph API:
POST /me/messenger_profile with { "get_started": { "payload": "GET_STARTED" } }
บอทรับ Postback นี้และแสดง Greeting อัตโนมัติ
ทดสอบ Facebook Messenger ครั้งสุดท้ายก่อน Go-Live
• ส่งข้อความครั้งแรก → ต้องได้รับ Greeting พร้อม Quick Replies (ไม่มีข้อความ "ทดสอบ")
• ถามเรื่องสินค้าที่มีลิงก์ใน Sheet → บอทตอบเป็น Generic Template Card
• ส่งรูปสินค้า → บอทตอบรับ + แสดงปุ่ม Quick Replies
• ทดสอบช่วงเวลาปิดบอท → บอทไม่ตอบ

4 หลัง Go-Live — Monitor และ Optimize

ติดตาม Vercel Logs 24 ชั่วโมงแรก
Vercel → Deployments → ล่าสุด → Logs (Runtime Logs)
ดูว่ามี error อะไรผิดปกติไหม เช่น Gemini timeout, Redis ล่ม, Signature ไม่ผ่าน
เช็ค /admin/unanswered ทุกวันช่วงแรก
เปิด https://line-bot-ai-theta.vercel.app/admin/unanswered
ดูคำถามที่บอทตอบไม่ได้ → นำไปเพิ่มใน Google Sheet ทันที
ช่วง 1–2 สัปดาห์แรกควรเช็คทุกวัน จนบอทตอบได้ครอบคลุม
ตรวจสอบ Gemini API Quota
Google AI Studio → ดู Usage ว่าใกล้ขีดจำกัดไหม
Gemini 2.5 Flash มี free tier ถ้าใช้มากขึ้นอาจต้องอัปเกรด Plan

5 สรุปลำดับขั้นตอน

1
ลบข้อความ "กำลังทดสอบ" จาก flex-cards.ts และ fb-webhook → commit + push
2
อัปเดต Google Sheet ให้ครบทุกสินค้าและข้อมูลที่ลูกค้าจะถาม
3
ตรวจ Env Vars ใน Vercel ครบทุกตัว
4
LINE: ตรวจ Webhook URL, ปิด Auto-reply, หา ADMIN_GROUP_ID
5
Facebook: ตรวจ Webhook, Submit App Review (ถ้ายังไม่ได้ทำ), ปิด Auto-reply ของ Page
6
ตั้งเวลาเปิด/ปิดบอท ใน /admin/schedule ให้ตรงเวลาทำการจริง
7
ทดสอบครั้งสุดท้าย ทั้ง LINE OA และ Facebook ก่อนประกาศให้ลูกค้าใช้
8
Go-Live! ประกาศให้ลูกค้าทราบ → Monitor Logs + เช็ค /admin/unanswered ทุกวัน
