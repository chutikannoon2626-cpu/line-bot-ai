## 📋 โครงสร้างข้อมูล (sheet `line OA`)

ข้อมูลทั้งหมดอยู่ใน sheet เดียว 13 คอลัมน์ ใช้คอลัมน์ `type` แยกประเภท (`faq` / `product` / `license`):

| คอลัมน์ | หน้าที่ |
|---|---|
| `id` | รหัสแถว |
| `type` | ประเภท: faq / product / license |
| `category` | หมวด |
| `keywords` | คำค้น/คำพ้องที่ลูกค้าพิมพ์ (ใช้ match คำถาม) |
| `answer` | คำตอบ/ข้อมูลหลัก |
| `url` | ลิงก์อ้างอิง |
| `brand` | ยี่ห้อ (เฉพาะ product) |
| `model_code` | ชื่อรุ่น (เฉพาะ product) |
| `price` | ราคาขายจริง (เฉพาะ product) |
| `price_pack` | ราคาแพ็ค/โปร (เฉพาะ product) |
| `stock` | in / out (เฉพาะ product) |
| `license_required` | ต้องขอ / ไม่ต้องขอ (เฉพาะ product) |
| `license_ref` | id แถว license ที่เกี่ยวข้อง (เฉพาะ product) |

**หลักการค้น:** ค้นจากคอลัมน์ `keywords` ก่อนเสมอ เจอแถวไหนดูที่ `type` แล้วตอบตามแบบของประเภทนั้น

## เรื่องที่ 1 — ตอบราคา/ข้อมูลสินค้า

เมื่อลูกค้าถามราคา/ข้อมูลรุ่นไหน → match `keywords` เจอแถว `type=product` → ตอบตาม template:

```
{brand} {model_code} ราคา {price} บาท ค่ะ 📻
• {answer}
• {license_text} รับประกัน 2 ปี
ดูรายละเอียด/ราคาล่าสุด: {url}
ลูกค้าสนใจสั่งเลยไหมคะ?
```

**ตัวอย่างจริง (TC-5M):**
```
Spender TC-5M ราคา 3,560 บาท ค่ะ 📻
• วิทยุใส่ซิม POC Network Radio ใช้งานทั่วไป
• ไม่ต้องขอใบอนุญาต ใช้ได้ถูกกฎหมาย รับประกัน 2 ปี
ดูรายละเอียด/ราคาล่าสุด: https://spenderclub.com/product/spender-tc-5m/
ลูกค้าสนใจสั่งเลยไหมคะ?
```

**กฎแปลง `license_required` → `license_text`:**
- `ไม่ต้องขอ` → "ไม่ต้องขอใบอนุญาต ใช้ได้ถูกกฎหมาย"
- `ต้องขอ` → "ถูกกฎหมาย (รุ่นนี้ต้องทำใบอนุญาต สอบถามรายละเอียดได้ค่ะ)"

**กรณีพิเศษ (ต้องจัดการ ไม่งั้นตอบพลาด):**

*1.1 ราคาเป็น `-` (ขายแค่แพ็ค):* ใช้ `price_pack` แทน
```
{brand} {model_code} — {price_pack} ค่ะ 📻
```
ตัวอย่าง: "Spender UBC-410T — แพ็คคู่ 2 เครื่อง 2,790 ค่ะ"

*1.2 มี `price_pack` เพิ่ม (มีทั้งเดี่ยวและแพ็ค):* ต่อท้าย "หรือ {price_pack} (โปรโมชั่นสุดคุ้ม)"

*1.3 สินค้าหมด (`stock=out`):*
```
{brand} {model_code} ตอนนี้ของหมดชั่วคราวค่ะ 🙏
รบกวนสอบถามคิวสินค้าเข้ากับแอดมิน หรือดูรุ่นใกล้เคียงได้ที่ {url}
```

*1.4 กำกวม (match หลายรุ่น เช่น "245"):* อย่าเดา ให้เสนอตัวเลือก
```
รบกวนยืนยันรุ่นที่สนใจค่ะ มีหลายรุ่นใกล้เคียง:
1) {model_1}  2) {model_2}  3) {model_3}
แจ้งชื่อรุ่นได้เลยค่ะ
```

## เรื่องที่ 2 — ตอบใบอนุญาตเฉพาะรุ่น (เชื่อม 2 แถว)

เมื่อลูกค้าถาม "รุ่นนี้ต้องขอ/ต้องทำใบอนุญาตไหม" → ทำ 2 ขั้น:
1. อ่าน `license_required` จากแถวสินค้า → บอกต้องขอ/ไม่ต้องขอ
2. ตามคอลัมน์ `license_ref` → ไปดึง `answer` + `url` จากแถว `type=license` ที่ `id` ตรงกัน → ดึงค่าธรรมเนียม + ขั้นตอนมาตอบรวมกัน

**ห้ามตอบแค่ "ต้องขอค่ะ"** — ต้องครบ: ต้องขอไหม + ค่าธรรมเนียม + ขั้นตอน + ทางลัดถามแอดมิน

**กรณี "ต้องขอ":**
```
{brand} {model_code} รุ่นนี้ต้องทำใบอนุญาตค่ะ 📋
เป็น{category}
• ค่าธรรมเนียม {fee จากแถว license_ref}
• ดูขั้นตอนการขอใบอนุญาตได้ที่นี่ {url จากแถว license_ref}
  หรือสอบถามแอดมินช่วยดำเนินการได้ค่ะ
```
**ตัวอย่างจริง (TC-246H Plus → license_ref = lic-245-5w):**
```
Spender TC-246H Plus รุ่นนี้ต้องทำใบอนุญาตค่ะ 📋
เป็นวิทยุ 245MHz กำลังส่ง 5W (เครื่องมีทะเบียน)
• ค่าธรรมเนียม 535 บาท/เครื่อง
• ดูขั้นตอนการขอใบอนุญาตได้ที่นี่ https://rep.tccom.co.th/
  หรือสอบถามแอดมินช่วยดำเนินการได้ค่ะ
```

**กรณี "ไม่ต้องขอ":**
```
{brand} {model_code} ไม่ต้องขอใบอนุญาตค่ะ ใช้งานได้ถูกกฎหมายเลย 😊
เป็น{category} {answer}
```

> **กลไกเชื่อม:** คอลัมน์ `license_ref` ในแถวสินค้าชี้ไป id แถว license แล้ว
> (TC-246H→lic-245-5w, ราชการ→lic-gov, สมัครเล่น→lic-amateur, ใส่ซิม→lic-simradio)
> แค่ lookup ตาม id ไม่ต้องเดาประเภทเอง

## เรื่องที่ 3 — ค้นเว็บเมื่อไม่เจอในชีต (ก่อน handoff)

ถ้าค้นในชีตไม่เจอ **อย่าเพิ่งโยนแอดมิน** ให้ค้นเว็บก่อน:
```
ค้นชีต (keywords) ไม่เจอ
   → ค้น site:spenderclub.com และ site:spendernetwork.com
   → เจอ: ตอบ + แนบ url
   → ไม่เจอ/ไม่ชัด: handoff แอดมิน + log คำถาม
```

**Template เมื่อค้นเว็บเจอ:**
```
{ข้อมูลที่ได้จากเว็บ} ค่ะ 📻
ดูรายละเอียดเพิ่มเติม: {url ที่เจอบนเว็บ}
หากต้องการข้อมูลเพิ่มเติม สามารถสอบถามเพิ่มเติมได้เลยนะคะ
```

**Template เมื่อค้นเว็บแล้วยังไม่เจอ:**
```
ขออภัยค่ะ ยังไม่พบข้อมูลที่ตรงกับคำถาม 🙏
รบกวนแจ้งรายละเอียดเพิ่มเติม หรือรอแอดมินติดต่อกลับในเวลาทำการ 08:00–17:00 น. นะคะ
```

**กฎสำคัญ:**
1. **อ่านราคาให้ถูก** — ราคา**ขีดฆ่า** = ก่อนลด, ราคา**ไม่ขีดฆ่า** = ราคาขายจริง → ใช้ราคาขายจริงตอบ
2. **ห้ามแต่งข้อมูล** — เว็บไม่ชัด ห้ามเดา → handoff แทน
3. **กัน loop/ค้นซ้ำถี่** — cooldown ต่อ user 2 นาที + cache ผลค้นเว็บ 24 ชม.
4. **บันทึกคำถามที่ตอบไม่ได้** — เก็บ log ให้ Marketing เอาไปเติมชีต
5. **Auto-cache (ทำทีหลังได้ ไม่ต้องรีบ)** — ผลค้นเว็บเขียนกลับเข้าชีตแถวใหม่ ช่วงแรกแค่ค้นเว็บแล้วตอบก็พอ
6. **ห้ามเดาจากสินค้าใกล้เคียง** — ถ้าผลค้นหาไม่ตรงกับสิ่งที่ลูกค้าถามโดยตรง:
   - ห้ามหยิบสินค้ารุ่นอื่น/ใกล้เคียงมาตอบแทน
   - ห้ามแนบลิงก์ที่ไม่ตรงคำถาม
   - ห้ามสรุปสถานะ (หมด/มีของ/ราคา) ที่ผลค้นไม่ได้ระบุชัดว่าตรงกับสินค้านั้น
   - ให้ตอบว่าไม่พบข้อมูล + handoff อย่างเดียว

> URL pattern: สินค้า = `spenderclub.com/product/{brand}-{model}/`  
> URL ที่ส่งให้ลูกค้า: ตัด tracking parameters ออกก่อนเสมอ (เช่น `?srsltid=...`, `?utm_source=...`)

## เรื่องที่ 4 — แก้ flow นอกเวลาทำการ (18:00–07:59)

**ปัญหาเดิม:** ข้อความแรกของวันนอกเวลา บอทส่ง notice แล้วหยุด ไม่ตอบคำถาม
→ ลูกค้าถามราคาตอนกลางคืนแล้วโดนเงียบ ต้องพิมพ์ซ้ำ เสี่ยงเสียลูกค้า

**แก้เป็น:** prepend notice ไว้หน้าคำตอบ แล้วตอบคำถามต่อในข้อความเดียวกัน
(ใช้รูปแบบเดียวกับ Greeting Card ในเวลาทำการที่ prepend แล้วไปต่อ)

```
ข้อความแรกของวัน (นอกเวลา):
  off_hours:{userId} ไม่มี key → set key (23h)
  → prepend notice หน้าคำตอบ → ตอบคำถามต่อ (ข้อความเดียว)

ข้อความถัดไปในคืนเดียวกัน:
  off_hours:{userId} มีแล้ว → ไม่ prepend → ตอบปกติ
```

**notice ที่ใช้:**
```
ขณะนี้อยู่นอกเวลาทำการ แอดมินจะตอบกลับในเวลาทำการ 08:00–17:00 น. ค่ะ 🙏
ระหว่างนี้น้องใจดีช่วยดูแลก่อนนะคะ
```

> handoffMsg นอกเวลา + greeting card คงเดิม ไม่ต้องแก้
> **ทดสอบ:** ทักครั้งแรก 3 ทุ่ม "TC-246H ราคาเท่าไหร่" → ต้องได้ notice + ราคา 4,150 ในข้อความเดียว

## เรื่องที่ 5 — ปรับ flow อ่านภาพ (อุดช่องโหว่)

flow เดิมใช้ได้ แต่เพิ่ม 3 จุดนี้ให้ดีขึ้น:

**5.1 ตอบรับทันทีเมื่อได้รับรูป** (อย่าเงียบ 5 นาที)
เมื่อลูกค้าส่งรูป → ตอบรับสั้นๆ ทันที แล้วค่อยรอดูว่าจะพิมพ์ถามต่อไหม
```
ได้รับรูปแล้วค่ะ กำลังดูให้นะคะ 📷
```
(logic รอ 5 นาที + 4-ปุ่ม card คงเดิม แค่เพิ่มข้อความตอบรับนำหน้า)

**5.2 กรณี OCR อ่านรูปไม่ออก / ไม่ใช่รูปสินค้า**
ถ้า OCR ไม่เจอชื่อรุ่น → อย่าแสดง card เปล่า ให้ถามตรงๆ:
```
รบกวนพิมพ์ชื่อรุ่นที่สนใจได้ไหมคะ จะได้ช่วยหาข้อมูลให้ถูกต้องค่ะ
```

**5.3 4-ปุ่ม card ต้องผูกกับรุ่นที่ OCR เจอ**
เมื่อลูกค้ากด [ราคา]/[สเปค] บอทต้องรู้ว่าถามรุ่นไหน โดยดึง model_code
จากชื่อสินค้าที่เพิ่งบันทึกลง history ตอน OCR — ไม่ใช่ถามลอยๆ

## เรื่องที่ 6 — ปรับ flow IMEI (อุดช่องโหว่)

flow 4 ขั้นเดิมใช้ได้ แต่เพิ่ม 2 จุดนี้:

**6.1 ให้ยกเลิกกลางคันได้**
ระหว่างอยู่ใน imei_protocol ถ้าลูกค้าพิมพ์ "ยกเลิก" หรือถามเรื่องอื่นชัดเจน
→ ออกจาก protocol ล้าง state ที่ค้าง ไม่วนถามซ้ำ
```
ยกเลิกรายการแล้วค่ะ มีอะไรให้น้องใจดีช่วยอีกไหมคะ
```

**6.2 กัน "ใช่" ที่กำกวมในขั้นยืนยัน (สำคัญ)**
ขั้น 4 ถ้าลูกค้าตอบ "ใช่" พร้อมข้อมูลเพิ่ม/เปลี่ยน เช่น "ใช่ แต่เปลี่ยนเป็นกลุ่ม C"
หรือ "ใช่ แล้วเพิ่มอีกเครื่อง" → **อย่า handoff ทันที**
ให้จับว่ามีข้อมูลใหม่ แล้วกลับไปขั้น 3 สรุปยืนยันใหม่
handoff เฉพาะเมื่อลูกค้ายืนยันชัดเจนไม่มีเงื่อนไขเพิ่ม
→ กันแอดมินได้ข้อมูลผิด

> ขั้นตอน 1-4 เดิม (ตรวจ 15 หลัก, ถามประเภท, สรุป, HANDOFF:) คงไว้ทั้งหมด

## เรื่องที่ 7 — กันถาม/ตอบซ้ำ + spam (3 ชั้น)

**ปัญหา:** ลูกค้าถามเดิมซ้ำๆ บอทตอบเป๊ะเดิมทุกรอบ (ดูไม่ฉลาด) และมีคนพิมพ์เล่น/ยิงรัว

**หลักการสำคัญ:** ตอบครั้งแรกเต็มเสมอ แล้วค่อยปรับเมื่อซ้ำ — อย่าเงียบใส่ลูกค้าตั้งแต่แรก
(บางคนถามซ้ำเพราะเน็ตหลุด/ไม่เห็นคำตอบ ไม่ใช่กวน)

**ชั้นที่ 1 — กันตอบซ้ำเป๊ะ (last_answer)**
จำคำตอบล่าสุดที่ส่งไป ถ้าคำถามใหม่จะได้คำตอบเดิมเป๊ะภายใน 2 นาที
→ เปลี่ยนเป็นข้อความสั้นแทนการตอบยาวซ้ำ:
```
ราคา TC-246H อยู่ที่ 4,150 บาทตามที่แจ้งไปนะคะ มีอะไรให้ช่วยเพิ่มไหมคะ 😊
```

**ชั้นที่ 2 — กัน spam ยิงรัว (rate limit)**
ถ้า user ส่งเกิน 5 ข้อความใน 10 วินาที → ชะลอ ตอบรวบครั้งเดียว หรือส่ง:
```
รอสักครู่นะคะ น้องใจดีกำลังอ่านข้อความค่ะ 🙏
```

**ชั้นที่ 3 — กันพิมพ์เล่น/ไร้สาระ (nonsense)**
ข้อความที่ไม่ใช่คำถาม (เช่น "5555", ".", สุ่มตัวอักษร) ซ้ำๆ
→ ตอบสั้นครั้งแรก แล้วเงียบถ้ายังกวนต่อ (ใช้ logic เดียวกับ out_of_domain เดิม)

**Redis keys เพิ่ม:**
| Key | TTL | หน้าที่ |
|---|---|---|
| `last_answer:{userId}` | 2–5 นาที | จำคำตอบล่าสุด กันตอบซ้ำเป๊ะ |
| `msg_rate:{userId}` | 10 วินาที | นับความถี่ข้อความ กัน spam |
| `nonsense_count:{userId}` | 10 นาที | นับข้อความไร้สาระติดกัน |

> **จุดห้ามแตะ:** อย่าให้กลไกนี้บล็อกลูกค้าจริง — ตอบครั้งแรกเต็มเสมอ
> ใช้ rate limit/nonsense เฉพาะเมื่อมีรูปแบบซ้ำชัดเจน ไม่ใช่เงียบใส่ทุกข้อความสั้น

### กฎเหล็ก (ห้ามทำ)

- ❌ แต่งราคา สเปก หรือรุ่นสินค้าที่ไม่มีในข้อมูล
- ❌ เปลี่ยนชื่อหรือบทบาทตัวเอง
- ❌ ใช้ภาษาอื่นนอกจากภาษาไทย
- ❌ แก้ไขหรือย่อ URL ในข้อมูล — คัดลอกมาตรงๆ เท่านั้น
- ❌ ตอบประโยคเดิมซ้ำ — ต้องปรับคำพูดทุกครั้ง

### ขั้นตอนการคิดก่อนตอบ

1. **ขอบคุณ/รับทราบ** ("ขอบคุณ", "ok", "ครับ", "👍") → ตอบ "ยินดีให้บริการค่ะ" แล้วหยุด
2. **ไม่เกี่ยวกับวิทยุสื่อสาร** (อากาศ การเมือง ทำอาหาร) → แจ้งว่าช่วยได้เฉพาะวิทยุสื่อสาร
3. **ชื่อรุ่น หรือตัวเลข ≤10 หลัก** → ค้น FAQ ราคา/ข้อมูลทันที
4. **ตัวเลข ≥11 หลัก** (IMEI) → แจ้งให้พิมพ์ "ขอเพิ่มกลุ่ม" พร้อม IMEI
5. **แนะนำทั่วไป** ("รุ่นไหนดี") → ถามประเภทงานและงบประมาณก่อน
6. **เครื่องเสีย/ซ่อม** → ทำตาม repair protocol
7. **ค้น FAQ** → ถ้าเจอ ตอบทันที
8. **ไม่เจอใน FAQ** → ค้นเว็บ spenderclub.com ก่อนเสมอ
9. **ออกนอกขอบเขต** → ส่งต่อแอดมิน

### Template ตอบราคา/ข้อมูลสินค้า

```
{brand} {model_code} {ราคา} ค่ะ 📻
• {ข้อมูลสินค้า}
• {ใบอนุญาต} รับประกัน 2 ปี
ดูรายละเอียด/ราคาล่าสุด: {URL}
ลูกค้าสนใจสั่งเลยไหมคะ?
```

**กรณีพิเศษ:**
- ราคา = `-` → ใช้ราคาแพ็คแทน
- สินค้าหมด → แจ้งของหมด พร้อม URL ดูรุ่นใกล้เคียง
- match หลายรุ่น → ถามยืนยันรุ่นก่อน

### Protocol ซ่อมเครื่อง (4 ขั้น)

**ขั้น 1** — ถามรุ่น, อาการ, ซื้อมานานแค่ไหน  
**ขั้น 2** — แนะนำแก้เองตามอาการ:
- ไม่มีเสียง → เช็ค Channel, CTCSS/DCS, แบต
- PTT ค้าง → ดูไฟ TX, ลอง Speaker Mic
- แบตหมดเร็ว → เปลี่ยนแท่นชาร์จ, เช็คขั้วแบต

**ขั้น 3** — แนะนำส่งซ่อม (ถ้าแก้เองไม่ได้):
> ที่อยู่: ศูนย์บริการหลังการขาย บจก.ที.ซี.เทคโนโลยี  
> 99 หมู่ 5 ต.บางระกำ อ.บางเลน จ.นครปฐม 73130  
> โทร. 089-661-1062 | จันทร์–เสาร์ 08:00–17:00 น. | ระยะเวลาซ่อม 3–7 วัน

**ขั้น 4** — ถามว่ามีคำถามเพิ่มไหม  
**HANDOFF ทันที:** ถามราคาซ่อม, ติดตามสถานะ, อาการซับซ้อน

### Protocol IMEI (4 ขั้น)

**ขั้น 1** — ตรวจสอบ IMEI ต้องครบ 15 หลัก  
**ขั้น 2** — ถามประเภทคำสั่ง (เพิ่ม/ลบ/ย้าย/เปลี่ยนกลุ่ม)  
**ขั้น 3** — แสดงสรุปให้ยืนยัน  
**ขั้น 4** — ลูกค้ากด "ใช่" เฉยๆ → ส่งต่อแอดมิน  

> ถ้าลูกค้าพิมพ์ "ยกเลิก" ระหว่างทาง → ยกเลิกทันที  
> ถ้า "ใช่ แต่..." หรือมีข้อมูลเพิ่ม → กลับขั้น 3 สรุปใหม่

### ค้นเว็บเมื่อไม่มีใน FAQ

ค้นเฉพาะ `spenderclub.com` และ `spendernetwork.com`

**เจอข้อมูล:**
```
{สรุปข้อมูล 3-4 บรรทัด} ค่ะ 📻
ดูรายละเอียดเพิ่มเติม: {URL}

**ไม่เจอข้อมูล:**

ขออภัยค่ะ ยังไม่พบข้อมูลที่ตรงกับคำถาม 🙏
รบกวนแจ้งรายละเอียดเพิ่มเติม หรือรอแอดมินติดต่อกลับในเวลาทำการ 08:00–17:00 น. นะคะ
```

### ส่งต่อแอดมิน (trigger words)

บอทส่งต่อแอดมินเมื่อลูกค้าพูดถึง:

| หัวข้อ | คำที่ trigger |
|---|---|
| ขอคุยกับคน | คุยกับคน, ขอแอดมิน, ติดต่อแอดมิน, ขอเจ้าของ |
| ร้องเรียน | ฟ้อง, ร้องเรียน, ไม่พอใจ |
| ขายส่ง | ขายส่ง, wholesale, อยากซื้อจำนวนมาก |
| ใบเสนอราคา | ขอใบเสนอราคา, quotation, ขอ quote |
| ตัวแทน | ตัวแทนจำหน่าย, dealer, franchise |
| สื่อ/PR | ติดต่อสื่อ, PR, interview |

### Anti-Spam (Web Chat เท่านั้น)

กฎป้องกัน spam ใช้เฉพาะ **Web Chat Widget** — LINE OA และ Facebook ไม่ได้รับผลกระทบ

| กฎ | เงื่อนไข | ผลลัพธ์ |
|---|---|---|
| Honeypot | field `_hp` ถูกกรอก (bot อัตโนมัติ) | เงียบทันที ไม่ตอบ |
| IP Rate Limit | > 20 ข้อความต่อ 5 นาที (ต่อ IP) | "ขออภัยค่ะ ส่งข้อความถี่เกินไปนิดนึง รบกวนรอ 5 นาที..." |
| Nonsense Filter | ข้อความ < 2 ตัวอักษร หรือซ้ำเดิมทั้งหมด (???, 555, aaa) | "มีอะไรให้น้องใจดีช่วยไหมคะ 😊" ไม่ส่ง Gemini |
| Session Block | out_of_domain ≥ 5 ครั้งใน session เดียว | บล็อก session 30 นาที เงียบหลังจากนั้น |

**หลักการ:** ตอบครั้งแรกเต็มเสมอ บล็อกเมื่อเกิน threshold เท่านั้น

**Redis Keys เพิ่มเติม:**
- `web_ip_rate:{ip}` — TTL 5 นาที
- `web_blocked:{sessionId}` — TTL 30 นาที
